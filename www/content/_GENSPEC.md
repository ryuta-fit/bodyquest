# BodyQuest 問題生成 共通仕様（厳守）

トレーナー／セラピスト向け学習アプリの問題を日本語で作成する。教科書的に正確な事実のみ。

## 出力ファイル形式（有効なJavaScript）
ファイルは必ず次の形で始め、配列を閉じて終わる（`<KEY>`=分野キー, `<ARR>`=同じキー）:

```
window.QUIZ_CONTENT = window.QUIZ_CONTENT || {};
window.QUIZ_CONTENT.<KEY> = (window.QUIZ_CONTENT.<KEY> || []).concat([
  { id:"<PREFIX>_301", cat:"...", diff:1, type:"mc", q:"...", choices:["A","B","C","D"], answer:0, exp:"...", tags:["..."] },
  ...
]);
```

## 各フィールド
- id: 指定の連番レンジ（3桁）。重複厳禁。
- cat: 指定カテゴリ一覧のいずれかに**完全一致**（教材連携のため）。全カテゴリを満遍なく。
- diff: 1基礎 / 2標準 / 3応用。
- type: "mc"(4択) / "tf"(正誤) / "fill"(穴埋め)。
- mc: choices=4要素、answer=正解index(0-3)。誤答も現実的で紛らわしく。
- tf: choices省略可。q を断定文にし、正しければ answer:0、誤りなら answer:1。
- fill: choices無し、answer="短い正解語"。表記ゆれは accept:["別表記"] を付与。
- exp: 1〜2文の解説。重要語を1箇所 `<b>…</b>` で強調し、理由や根拠を一言添える。
- tags: 1〜3個（任意）。

## 文字列の安全規則（JS破壊防止）
- 文字列内に二重引用符( " )やバックスラッシュ( \ ) を使わない。強調・引用は日本語かぎ括弧「」。
- 掛け算は「×」、割り算は「÷」、角度は「度」。
